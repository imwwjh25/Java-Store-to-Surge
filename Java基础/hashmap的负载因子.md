在 HashMap 中，**负载因子（Load Factor）是一个预先设定的阈值**，用于决定哈希表何时需要进行扩容（resize），它并非通过计算得到，而是 JDK 中默认定义的一个常量（默认值为 `0.75`）。

不过，负载因子与哈希表的**扩容时机**直接相关，其核心作用是通过以下公式判断是否需要扩容：





```plaintext
扩容阈值（threshold）= 哈希表当前容量（capacity） × 负载因子（loadFactor）
```

当 HashMap 中存储的键值对数量（size）**超过扩容阈值**时，哈希表会触发扩容（容量翻倍），以减少哈希冲突，保证查询效率。

### 具体说明：

1. **负载因子的默认值**HashMap 的默认负载因子为 `0.75`，这是 JDK 设计者在时间和空间成本之间权衡的结果：

    - 若负载因子过大（如 1.0）：哈希表扩容频率低，节省内存空间，但哈希冲突概率高，链表 / 红黑树会变长，查询效率下降（时间成本增加）。
    - 若负载因子过小（如 0.5）：哈希表扩容频繁，内存占用增加，但哈希冲突少，查询效率高（时间成本降低）。

2. **负载因子的自定义**可以在创建 HashMap 时通过构造函数指定负载因子（默认容量为 16）：





   ```java
   // 初始容量为 32，负载因子为 0.8
   Map<String, Integer> map = new HashMap<>(32, 0.8f);
   ```



但通常不建议修改，除非明确了解业务场景对时间 / 空间的需求。

3. **扩容阈值的动态变化**初始状态下，HashMap 容量为 16，扩容阈值 = 16 × 0.75 = 12。当插入第 13 个键值对时，触发扩容，容量变为 32，新的阈值 = 32 × 0.75 = 24，以此类推。

### 总结：

负载因子是 HashMap 中**预先设定的常量**（默认 0.75），并非计算得出。它通过与当前容量相乘得到 “扩容阈值”，决定哈希表何时扩容，其值的大小直接影响 HashMap 的内存占用和查询性能。
