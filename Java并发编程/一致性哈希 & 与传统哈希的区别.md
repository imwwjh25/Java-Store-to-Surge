#### （1）一致性哈希是什么？

解决传统哈希在「节点扩容 / 缩容」时大量数据迁移的问题（如分布式缓存集群）。

#### （2）实现原理：

1. 构建一个**环形哈希空间**（0~2³²-1）；
2. 对集群节点（IP / 主机名）做哈希，映射到环上；
3. 对数据 Key 做哈希，沿环顺时针找最近的节点，即为数据归属节点。

#### （3）与传统哈希的区别：

|     维度     |  传统哈希（如 `hash(key)%N`）   |        一致性哈希        |
| :----------: | :-----------------------------: | :----------------------: |
| 节点变化影响 | 节点数 N 变，所有数据需重新映射 | 仅影响相邻节点，迁移量小 |
|   负载均衡   |      依赖哈希函数，易失衡       |  可通过虚拟节点均衡负载  |
|   适用场景   |    静态节点（如单机哈希表）     | 动态节点（如分布式缓存） |
