### 一、硬件层面：原子性的根基

硬件通过**原子指令、总线锁 / 缓存锁、缓存一致性协议**，直接保证单指令或内存访问的原子性。

#### 1. 原子指令（核心）

CPU 指令集提供专用原子指令，单指令周期内完成读写，不可被中断。

- x86 架构

    - ```
    LOCK
    ```

      前缀：配合普通指令（如

      ```
      add
      ```

      、

      ```
      xchg
      ```

      ），锁定内存访问。

      asm











    ```
    lock add [counter], 1  ; 原子自增
    ```



- `XCHG`：原子交换寄存器与内存值。

- `CMPXCHG`：比较并交换（CAS），实现无锁更新。

- `XADD`：原子加并返回旧值。



- ARM/RISC-V 架构

    - ```
    LDREX/STREX
    ```

      （LL/SC）：加载链接 / 存储条件。

        1. `LDREX`：加载值并标记地址为 “独占”。
        2. `STREX`：仅在标记未失效时写入，否则失败重试。





#### 2. 总线锁 vs 缓存锁

- 总线锁（早期） ：CPU 发```LOCK#```

  信号，锁定系统总线，其他 CPU 无法访问内存。

    - 缺点：粒度大、性能差，阻塞所有内存访问。



- 缓存锁（现代主流）

  ：基于 MESI 缓存一致性协议 ，仅锁定目标缓存行。

    - MESI 状态：Modified（修改）、Exclusive（独占）、Shared（共享）、Invalid（无效）。
    - 执行 `LOCK` 指令时，CPU 将缓存行设为 **Exclusive**，其他核心缓存行设为 **Invalid**，保证独占访问。



#### 3. 缓存一致性（MESI）

多核下，MESI 保证各核心缓存与主存一致，是缓存锁的基础。

- 核心 A 修改缓存行 → 标记为 Modified → 通知其他核心该缓存行失效。
- 其他核心读取时，必须从主存或核心 A 同步最新值。

### 二、操作系统层面：基于硬件的原子封装

OS 利用硬件原子能力，实现**中断屏蔽、自旋锁、信号量、原子类型**，为上层提供原子性保障。

#### 1. 单核系统：中断屏蔽（关中断）

- 原理：内核执行```cli```（关中断）→ 临界区 →```sti```开中断），期间不响应任何中断（包括调度）。









  ```
  local_irq_disable();  // 关中断
  // 临界区（如修改进程链表）
  local_irq_enable();   // 开中断
  ```



- 适用：单核内核原语（如进程调度、信号处理），用户态不可用。

#### 2. 多核系统：自旋锁（Spinlock）

- 基于硬件 CAS 实现，线程循环尝试获取锁，不阻塞、不调度。










  ```
  spin_lock(&lock);    // 原子 CAS 尝试加锁
  // 临界区
  spin_unlock(&lock);  // 释放锁
  ```



- 适用：多核内核短临界区，避免上下文切换开销。

#### 3. 原子类型与系统调用

- Linux 内核：

  ```atomic_t```类型，封装硬件原子指令。```atomic_t v = ATOMIC_INIT(0); atomic_inc(&v);      // 原子自增```



- 系统调用：如 `sys_futex`，基于 CAS 实现用户态锁与等待队列。

#### 4. 信号量 / 互斥锁（Mutex）

- 基于原子操作 + 等待队列，长临界区时阻塞线程，减少 CPU 空转。

- 原理：

    1. 原子检查锁状态 → 占用则加入等待队列并阻塞。
    2. 释放锁时，原子唤醒队列首线程。



------

### 三、软硬件协作流程（以 i++ 为例）

1. **用户态**：`std::atomic<int> a; a++;`

2. **编译器**：翻译为 `lock xadd [a], 1`（x86）。

3. 硬件

   ：

    - 锁定缓存行 → 读取 a → 加 1 → 写回 → 释放锁。
    - MESI 保证其他核心缓存失效。



4. **OS**：无需介入，硬件直接保证原子性。

------

### 四、关键对比









| 层面           | 核心机制               | 适用场景           | 性能 |
| -------------- | ---------------------- | ------------------ | ---- |
| **硬件**       | 原子指令、缓存锁、MESI | 单指令 / 内存访问  | 极高 |
| **OS（单核）** | 中断屏蔽               | 内核短临界区       | 高   |
| **OS（多核）** | 自旋锁、Mutex          | 多核并发、长临界区 | 中高 |

------

### 总结

- **硬件**：提供**原子指令、缓存锁、MESI**，是原子性的底层保障。
- **操作系统**：基于硬件封装**中断屏蔽、自旋锁、Mutex、原子类型**，适配不同场景。
- 上层语言（Java/C++）的原子类（`AtomicInteger`/`std::atomic`）最终都依赖这两层能力。
