**填充对齐的核心目的是提升 CPU 内存访问效率**，这是 JVM 基于硬件架构的优化策略。

### 关键原因



- CPU 访问内存时，并非单个字节读取，而是按**缓存行（通常 64B）** 或**机器字长（32 位 4B、64 位 8B）** 批量读取。
- 若对象大小不满足对齐要求（如 64 位 JVM 默认按 8B 对齐），可能导致一个对象跨多个缓存行，CPU 需读取两次，效率大幅降低。

### 空实例的填充示例（64 位 JVM 开启压缩指针）



- 空实例对象头占 12B，不满足 8B 的整数倍对齐要求。
- JVM 会自动填充 4B 空白数据，使总大小达到 16B（8B 的 2 倍），确保对象存储在单个缓存行内，提升访问速度。
