匿名类是 Java 中一种没有显式命名的局部内部类，通常用于**快速创建一个类的实例，且只需要使用一次**的场景。它可以简化代码，避免为临时使用的类单独定义类名。

### 匿名类的核心特点：

1. **没有类名**：定义时直接通过父类或接口创建实例，无需单独声明 `class`。
2. **一次性使用**：通常只创建一个实例，用完即弃，不适合重复使用。
3. **继承或实现**：必须继承一个父类，或实现一个接口（且只能有一个父类 / 接口）。
4. **作用域有限**：定义在方法内部或表达式中，仅在当前作用域有效。

### 语法格式：







```java
// 基于父类的匿名类
父类类型 变量名 = new 父类类型() {
    // 重写父类方法或定义新方法
    @Override
    public void 方法名() { ... }
};

// 基于接口的匿名类（接口不能实例化，匿名类本质是实现接口的子类）
接口类型 变量名 = new 接口类型() {
    // 实现接口的所有抽象方法
    @Override
    public void 方法名() { ... }
};
```

### 示例：

#### 1. 实现接口的匿名类






```java
// 定义一个接口
interface Greeting {
    void sayHello();
}

public class Main {
    public static void main(String[] args) {
        // 匿名类实现 Greeting 接口
        Greeting greeting = new Greeting() {
            @Override
            public void sayHello() {
                System.out.println("Hello, 匿名类！");
            }
        };
        greeting.sayHello(); // 输出：Hello, 匿名类！
    }
}
```

#### 2. 继承父类的匿名类








```java
// 定义一个父类
class Animal {
    public void cry() {
        System.out.println("动物叫");
    }
}

public class Main {
    public static void main(String[] args) {
        // 匿名类继承 Animal 并重写方法
        Animal cat = new Animal() {
            @Override
            public void cry() {
                System.out.println("猫叫：喵喵~");
            }
        };
        cat.cry(); // 输出：猫叫：喵喵~
    }
}
```

### 常见使用场景：

- **简化回调逻辑**：如多线程中 `Runnable` 接口的实现（`new Thread(new Runnable() { ... }).start()`）。
- **临时实现接口**：如 GUI 编程中的事件监听器（`button.addActionListener(new ActionListener() { ... })`）。
- **避免定义冗余类**：当某个类仅需使用一次时，用匿名类减少代码量。

### 注意点：

- 匿名类中不能定义构造方法（因没有类名）。
- 匿名类可以访问外部类的成员变量，但访问局部变量时，局部变量必须是 `final` 或事实上的 final（Java 8+ 可省略 `final` 关键字，但值不能修改）。

简单说，匿名类就是 “即用即弃” 的临时类，核心价值是**在不增加类定义的前提下，快速实现一个类的功能**。
