HTTP 状态码是服务器对客户端请求的响应状态标识，分为五大类，每类包含不同的具体状态码，用于指示请求处理的结果。以下是详细分类及常见状态码：

### **一、信息性状态码（1xx）**

表示服务器已接收请求，正在处理中，是临时响应，不包含响应体。

- **100 Continue**：客户端应继续发送请求的剩余部分（常用于 POST 大数据时，服务器确认可接收后返回）。
- **101 Switching Protocols**：服务器同意客户端的协议切换请求（如从 HTTP 切换到 WebSocket）。

### **二、成功状态码（2xx）**

表示请求已被服务器成功接收、理解并处理。

- **200 OK**：请求成功，响应体包含请求结果（最常见的成功状态）。
- **201 Created**：请求成功且创建了新资源（如 POST 提交表单创建用户后返回），响应头通常包含`Location`指示新资源地址。
- **204 No Content**：请求成功，但无响应体（如 DELETE 删除资源后，无需返回内容）。
- **206 Partial Content**：客户端请求部分资源（带`Range`头），服务器成功返回部分内容（如断点续传）。

### **三、重定向状态码（3xx）**

表示客户端需要进一步操作才能完成请求，通常需要跳转。

- **301 Moved Permanently**：资源永久迁移到新 URL，客户端应使用新 URL（浏览器会缓存新地址）。
- **302 Found**：资源临时迁移到新 URL，客户端应继续使用原 URL（浏览器不缓存新地址，常用于临时跳转）。
- **304 Not Modified**：客户端请求的资源未修改，可使用本地缓存（配合`If-Modified-Since`等条件请求头使用）。
- **307 Temporary Redirect**：与 302 类似，但严格要求客户端使用原请求方法（如 POST 不能转为 GET）。
- **308 Permanent Redirect**：与 301 类似，同样要求客户端保持原请求方法。

### **四、客户端错误状态码（4xx）**

表示请求存在错误，服务器无法处理。

- **400 Bad Request**：请求格式错误（如参数无效、JSON 格式错误）。
- **401 Unauthorized**：请求需要身份验证（如未登录，响应头通常包含`WWW-Authenticate`说明验证方式）。
- **403 Forbidden**：服务器拒绝请求（已验证身份，但无权限访问）。
- **404 Not Found**：请求的资源不存在（最常见的错误状态）。
- **405 Method Not Allowed**：请求方法不被允许（如用 POST 访问仅支持 GET 的接口），响应头`Allow`会列出允许的方法。
- **406 Not Acceptable**：服务器无法返回客户端请求的格式（如客户端要求`application/json`，但服务器只能返回`text/html`）。
- **408 Request Timeout**：客户端请求超时。
- **409 Conflict**：请求与资源当前状态冲突（如修改资源时，资源已被他人更新）。
- **413 Payload Too Large**：请求体过大，服务器无法处理。
- **414 URI Too Long**：请求的 URL 过长。
- **429 Too Many Requests**：客户端请求过于频繁（常用于限流）。

### **五、服务器错误状态码（5xx）**

表示服务器处理请求时发生内部错误。

- **500 Internal Server Error**：服务器内部错误（如代码 bug、数据库故障）。
- **501 Not Implemented**：服务器不支持请求的功能（如未实现的 HTTP 方法）。
- **502 Bad Gateway**：服务器作为网关 / 代理时，收到上游服务器的无效响应（如反向代理后端服务故障）。
- **503 Service Unavailable**：服务器暂时无法处理请求（如维护中，可通过`Retry-After`头告知客户端重试时间）。
- **504 Gateway Timeout**：服务器作为网关 / 代理时，上游服务器响应超时。
- **505 HTTP Version Not Supported**：服务器不支持请求的 HTTP 版本（如客户端用 HTTP/2，服务器仅支持 HTTP/1.1）。

### 总结

状态码的首位数字代表类别，便于快速判断问题方向：

- 1xx：临时响应；2xx：成功；3xx：重定向；4xx：客户端错误；5xx：服务器错误。
  实际开发中，需根据具体状态码排查问题（如 4xx 检查请求参数或权限，5xx 检查服务器日志）。
