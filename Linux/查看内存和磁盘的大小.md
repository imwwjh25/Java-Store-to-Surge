
### 一、查看内存大小：`free` 命令



`free` 用于显示系统的物理内存、交换内存（Swap）的使用情况，核心是看 **总内存、已用内存、可用内存**。

#### 1. 基础用法（常用参数 `-h`，人性化显示单位）




```
free -h  # -h：自动转换为GB/MB等易读单位（默认是KB，可读性差）
```



#### 2. 输出结果解读（示例）





```plaintext
              total        used        free      shared  buff/cache   available
Mem:           15Gi       2.3Gi        11Gi       152Mi       1.7Gi        12Gi
Swap:          19Gi          0B        19Gi
```



- Mem ：物理内存（重点关注）

    - `total`：总物理内存（如 15Gi = 15GB）
    - `used`：已被应用程序直接使用的内存
    - `free`：完全空闲的内存
    - `available`：**实际可被新程序使用的内存**（= free + buff/cache 中可回收部分，最有参考价值）

- **Swap**：交换内存（用磁盘模拟的内存，速度慢，一般用于内存不足时应急）

#### 3. 补充：查看更详细的内存占用（`top` 或 `htop`）



若想知道 “哪些进程占内存多”，用 `top` 命令（实时刷新）：

```
top  # 按 Shift+M 可按内存占用排序，直观看到内存大户
```



### 二、查看磁盘大小：`df` 命令



`df` 用于显示磁盘分区的 **总容量、已用容量、可用容量及挂载点**，核心是看各分区的空间使用情况。

#### 1. 基础用法（常用参数 `-h`，人性化显示；`-T` 显示分区类型）


```
df -hT  # -h：易读单位；-T：显示分区格式（如 ext4、xfs 等）
```



#### 2. 输出结果解读（示例）


```plaintext
Filesystem     Type      Size  Used Avail Use% Mounted on
/dev/sda1      ext4      9.8G  3.2G  6.1G  35% /boot
/dev/mapper/cl-root xfs      47G   12G   35G  26% /
/dev/mapper/cl-home xfs      40G    5G   35G  13% /home
```



- **Filesystem**：磁盘分区名（如 `/dev/sda1`）
- **Type**：分区格式（如 ext4、xfs，不同格式功能略有差异）
- **Size**：分区总容量（如 9.8G = 约 10GB）
- **Used/Avail**：已用容量 / 可用容量
- **Use%**：使用率（若超过 90% 需警惕，可能导致磁盘满、程序报错）
- **Mounted on**：分区挂载的目录（如 `/boot` 是启动分区，`/` 是系统根目录）

#### 3. 补充：查看指定目录的磁盘占用（`du` 命令）



若想知道 “某个目录下的文件占多少空间”，用 `du` 命令（如查看 `/home` 目录）：


```
du -sh /home  # -s：只显示总大小；-h：易读单位，输出如 "5.0G /home"
```



### 总结



| 需求                    | 核心命令 | 常用参数     | 关键看什么                        |
| ----------------------- | -------- | ------------ | --------------------------------- |
| 查看内存大小 / 使用     | `free`   | `-h`         | 物理内存的 `total` 和 `available` |
| 查看磁盘分区大小 / 使用 | `df`     | `-hT`        | 各分区的 `Size`、`Avail`、`Use%`  |
| 查看目录 / 文件占用     | `du`     | `-sh 目录名` | 目录的总占用空间                  |

这三个命令是 Linux 运维和日常使用的 “必背” 命令，记住 `free -h` 和 `df -hT` 就能快速搞定内存、磁盘的基础查看需求。
